# Design Doc: 前端全面改版 — 数据仪表盘风格

## 概述
**需求来源**：PM 反馈现有前端"写的和屎一样"，要求全面重做
**设计日期**：2026-04-27
**状态**：草稿

## 问题定义

现有前端是一个 Tailwind 默认深色主题堆砌的毛坯房。三个核心问题：

1. **视觉无辨识度** — 全是 `slate-800/900`，没有品牌感，没有视觉焦点。`ModuleNode` 的复杂度背景色三个分支几乎一样（`bg-slate-800/90` vs `bg-slate-800/95`），复杂度可视化形同虚设
2. **信息层级扁平** — ProjectList 是表单+列表，ProjectView 是全屏图+侧边栏，没有统计概览、没有引导用户注意力的设计
3. **交互反馈弱** — 没有页面过渡、组件动画、加载骨架屏。操作感像填 Excel

**目标**：重做为数据仪表盘风格（参考 Grafana、Datadog、GitHub Insights），信息密集但层次分明，让非技术 PM 一眼看到项目健康度。

## 方案设计

### 推荐方案：自定义设计系统 + 组件重构

**选择理由**：应用体量小（2 页面、~6 组件），不需要引入 shadcn/ui 等组件库。通过 Tailwind v4 CSS 变量建立设计 token，手写组件最灵活、bundle 最小。用 framer-motion 做动画——体积可控（tree-shake 后 ~15KB），效果拉满。

#### 一、设计系统（Design Tokens）

通过 CSS 变量定义，Tailwind v4 原生支持 `@theme` 注入。

**配色**：

| Token | 色值 | 用途 |
|-------|------|------|
| `--bg-base` | `#06080d` | 页面底色（接近纯黑，微蓝调） |
| `--bg-surface` | `#0d1117` | 一级卡片/面板（GitHub Dark 同色系） |
| `--bg-elevated` | `#161b22` | 二级表面/弹窗 |
| `--bg-overlay` | `#1c2333` | hover/active 状态、浮层 |
| `--border-default` | `#21262d` | 默认边框 |
| `--border-emphasis` | `#363b44` | 强调边框 |
| `--text-primary` | `#e6edf3` | 主文字 |
| `--text-secondary` | `#8b949e` | 次要文字 |
| `--text-muted` | `#484f58` | 禁用/提示文字 |
| `--accent-cyan` | `#22d3ee` | 主色调——数据高亮、主按钮、链接 |
| `--accent-purple` | `#a78bfa` | 辅助色——次要操作、辅助数据 |
| `--status-green` | `#3fb950` | 健康/简单 |
| `--status-yellow` | `#d29922` | 警告/中等 |
| `--status-red` | `#f85149` | 危险/复杂 |
| `--status-blue` | `#58a6ff` | 信息/进行中 |

**排版**：

| 用途 | 字号 | 字重 | 字体 |
|------|------|------|------|
| 统计大数字 | 2rem / 32px | 700 | `"JetBrains Mono", monospace` |
| 页面标题 | 1.5rem / 24px | 600 | `"Inter", system-ui, sans-serif` |
| 段落标题 | 1rem / 16px | 600 | Inter |
| 正文 | 0.875rem / 14px | 400 | Inter |
| 辅助文字 | 0.75rem / 12px | 400 | Inter |
| 代码/数据 | 0.8125rem / 13px | 400 | JetBrains Mono |

**间距**：4px 基础单位，常用 8/12/16/24/32px。

**圆角**：`sm: 6px`、`md: 8px`、`lg: 12px`、`xl: 16px`。

**动画曲线**：`cubic-bezier(0.16, 1, 0.3, 1)`（快进慢出），持续时间 150-300ms。

```
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
├─────────────────────────────────────────────────────────────┤
│  项目卡片网格（responsive grid: 1~3 列）：                    │
│  ┌───────────────────────┐  ┌───────────────────────┐       │
│  │ tvbox-aggregator      │  │ my-blog               │       │

│  │ (绿60% 黄30% 红10%)    │  │ (绿90% 黄10%)          │       │
│  │                       │  │                       │       │

│  └───────────────────────┘  └───────────────────────┘       │

│  ┌───────────────────────┐                                  │
│  │  ＋ 添加新项目          │  ← 虚线边框的占位卡片             │

│  └───────────────────────┘                                  │
└─────────────────────────────────────────────────────────────┘
```

**交互细节**：
- 项目卡片 hover：边框变 cyan，整卡微上移 2px + 阴影扩大
- "添加项目" 点击后弹出 Modal（不是内联表单），Modal 有毛玻璃背景
- 空状态：大图标 + "还没有项目，从这里开始" CTA
- 概览统计数字用 `JetBrains Mono` 大字号，数字变化时有数值滚动动画
- 复杂度分布条：三色横向堆叠条形图（绿/黄/红），纯 CSS 实现，不需要图表库

**数据来源**：
- 概览统计 = 所有项目 stats 求和（前端聚合）
- 项目卡片数据 = `GET /api/projects` 改为返回 `ProjectWithStats[]`

#### 三、ProjectView → 项目分析仪表盘

**当前**：56px 高的头 + 全屏 ReactFlow + 侧边栏
**改为**：多层信息仪表盘

**布局**：

```
┌─────────────────────────────────────────────────────────────┐
│  顶栏：← 返回  ·  tvbox-aggregator  ·  [重新扫描]           │
├─────────────────────────────────────────────────────────────┤
│  指标条（4 个 stat card + 扫描状态）：                        │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐  ● 已扫描     │
│  │ 8 模块  │ │ 95 文件 │ │ 12k 行  │ │ 42 复杂度│  2小时前    │
│  └────────┘ └────────┘ └────────┘ └────────┘               │
├─────────────────────────────────────────────┬───────────────┤
│                                             │  模块详情      │
│            模 块 地 图                        │  ──────────── │
│        （ReactFlow 可视化）                   │  stats/deps   │
│                                             │  /files 三个   │
│     节点颜色按复杂度渐变                       │  tab 页        │
│     边粗细按依赖权重                           │               │
│     背景网格 + 渐变底色                        │               │
│                                             │               │
├─────────────────────────────────────────────┴───────────────┤
│  底部无需额外区域，信息集中在图+侧边栏                         │
└─────────────────────────────────────────────────────────────┘
```

**指标条改进**：
- 每个 stat card 带一个小图标 + 趋势指示（如 "↑ 比上次多 3 个模块"，后续迭代加）
- 当前先做静态数字显示 + 扫描状态徽章
- 扫描进行中时，进度条横贯整个指标条底部（细条，cyan 色，动画）

**模块地图改进**：
- 背景：从 `#06080d` 到 `#0d1117` 的径向渐变（视觉中心亮一点）
- 网格：更暗的网格线 `#161b22`，间距 24px，点状而非线状
- 控件（zoom/pan/fit）：重新样式化为圆角按钮组
- MiniMap：半透明悬浮在右下角，圆角 + 边框

**模块节点（ModuleNode）全面重做**：

```
┌─────────────────────────────────┐
│  ● src/services                 │  ← 左侧圆点 = 复杂度色
│                                 │
│  12 文件  ·  2.4k 行  ·  复杂 67 │  ← 一行三个指标
│  ████████████░░░░ 67%           │  ← 复杂度进度条
└─────────────────────────────────┘
```

- 背景：根据复杂度有微妙的颜色倾向
  - 简单（<30）: `#0d1117` + 绿色左边框 `border-l-3 border-green-500`
  - 中等（30-59）: `#0d1117` + 黄色左边框
  - 复杂（≥60）: `#0d1117` + 红色左边框 + 微弱的红色 glow
- 选中态：cyan 边框 + `box-shadow: 0 0 20px rgba(34, 211, 238, 0.15)`
- hover：边框变亮 + 微上移
- 尺寸仍根据 `relativeSize` 动态计算

**边（Edges）改进**：
- 低权重（1-2）：`#21262d`，1px，虚线
- 中权重（3-5）：`#363b44`，2px，实线
- 高权重（>5）：`--accent-cyan` 半透明，3px，animated
- hover 边时高亮该边 + 两端节点

#### 四、ModuleDetail → 带 Tab 的详情面板

**当前**：平铺的 stats + deps + files
**改为**：三个 Tab 页切换

| Tab | 内容 | 视觉 |
|-----|------|------|
| 概览 | 大数字统计卡（文件数/行数/复杂度得分） + 复杂度仪表盘（半圆进度条） | 数字用 `JetBrains Mono` 加大显示 |

**面板头部**：
- 模块名 + 复杂度徽章
- 关闭按钮（X 图标，不是 ✕ 文字）
- 面板从右侧滑入（framer-motion `AnimatePresence`）

#### 后端

- **修改接口**：

  | 接口 | 方法 | 变更 | 原因 |
  |------|------|------|------|
  | `/api/projects` | GET | 返回 `ProjectWithStats[]` 而非 `Project[]` | 首页项目卡片需要显示 stats，避免 N+1 请求 |

- **核心逻辑**：在 list 查询中 LEFT JOIN modules 和 dependencies 表，用子查询聚合 stats。一次查询搞定。

- **数据模型变更**：无。

---

### 备选方案：引入 shadcn/ui

**放弃理由**：

1. 应用只有 2 页面 ~6 组件，引入组件库的配置成本（安装 + 主题配置 + 覆写样式）高于手写收益
2. 仪表盘需要大量自定义样式（渐变背景、glow 效果、复杂度配色），shadcn 的默认组件反而会成为要绕开的限制
3. 唯一真正有用的是 Dialog/Modal 组件，但手写一个带 framer-motion 动画的 Modal 不超过 50 行

### 备选方案：只改样式不重构组件

**放弃理由**：

1. 当前组件结构就有问题——ModuleNode 的复杂度可视化是假的（三个分支同色），不改结构修不了
2. ProjectList 缺少 stats 数据流，只换 CSS 无法实现仪表盘首页
3. 侧边栏改 Tab 需要组件状态重构，不是样式层面的事

## 影响评估

- **涉及模块**：
  - 前端：全部页面和组件（ProjectList、ProjectView、ModuleMap、ModuleNode、ModuleDetail），新增 Modal、StatsCard、SkeletonCard 组件
  - 后端：仅 `projects.ts` 路由的 list 端点
  - 样式：`index.css` 新增设计 token
  - 依赖：新增 `framer-motion`

- **风险点**：
  - ReactFlow 样式覆写可能需要调试——它自带 CSS，我们的自定义主题需要精确覆盖
  - `JetBrains Mono` 字体加载可能影响首屏，需要用 `font-display: swap` + 预加载
  - 后端 list 接口加 JOIN 在项目数少的情况下无性能问题，但如果项目数上百需要加索引

- **技术债**：
  - 本次不加路由库（仍靠 zustand 的 currentProjectId 切换页面）——后续功能多了必须上 react-router，标记为已知债务
  - 颜色 token 目前通过 CSS 变量定义，未抽象为 Tailwind plugin——规模小时够用，规模大了要迁移

## 新增依赖

| 包 | 版本 | 理由 | 大小 |
|----|------|------|------|
| `framer-motion` | ^12 | 页面过渡、面板滑入、数字动画 | ~15KB gzipped (tree-shaken) |

**不引入的**：
- `recharts`/`@nivo` — 复杂度分布条用纯 CSS 实现，不需要图表库
- `shadcn/ui` — 理由见备选方案分析
- `react-router` — 标记为技术债，本次不引入

## 验收标准

1. **ProjectList** 打开后能看到概览统计条 + 项目卡片网格，每个卡片显示 stats 和复杂度分布条
2. **ProjectView** 有指标条、重做后的模块节点（左边框颜色 + 进度条 + 正确的复杂度可视化）
3. **ModuleDetail** 以侧边 Tab 面板形式从右侧滑入，三个 Tab 可切换
4. **添加项目** 弹出 Modal 而非内联表单
5. 所有页面切换有 framer-motion 过渡动画
6. 异步加载点有骨架屏，不出现裸 "loading..." 文字
7. 设计 token 通过 CSS 变量定义在 `index.css`，所有组件通过变量引用颜色，不直接写色值
8. 后端 `GET /api/projects` 返回带 stats 的数据，前端不需要为每个项目单独请求
