name: review
description: 手动触发代码审查。派发 code-reviewer sub-agent 独立审查代码变更，防止自审。
---

[任务]
    收集最近的代码变更，派发给 code-reviewer sub-agent 进行独立审查。
    审查覆盖：架构对齐、代码质量、前后端一致性、安全风险。

[核心原则]
    - **独立审查**：必须通过 sub-agent 执行，主 agent 不能自己审自己
    - **设计对齐**：审查的第一标准是代码是否与 Design-Doc 一致
    - **前后端一致**：前端展示逻辑和后端数据逻辑必须匹配

[审查维度清单]
    **必须检查**：
    - 架构对齐：代码实现是否与 Design-Doc 描述一致？
    - 接口契约：前后端接口是否匹配（字段名、类型、格式）？
    - 安全基线：有没有 SQL 注入、XSS、硬编码密钥等基本安全问题？
    - 错误处理：系统边界的错误是否被正确处理？

    **尽量检查**：
    - 代码可读性：命名是否自解释？逻辑是否清晰？
    - 重复代码：有没有可以复用的重复逻辑？
    - 性能隐患：有没有明显的 N+1 查询、内存泄漏等问题？

[工作流程]
    [审查执行]
        目的：独立审查代码质量

        第一步：收集变更
            获取最近的代码变更（git diff 或指定范围）
            读取对应的 Design-Doc 作为审查基准

        第二步：派发审查
            将变更和 Design-Doc 传递给 code-reviewer sub-agent
            等待审查结果

        第三步：整理报告
            将 sub-agent 的审查结果整理为 Review-Report
            加载 templates/review-report-template.md 获取模板格式
            保存到 docs/

        第四步：呈现结果
            向用户展示审查报告
            有问题 → 建议 /build 修复
            无问题 → 确认可进入下一轮迭代
